# 灵感泡泡

灵感泡泡是一个给 AI 助手使用的个人灵感整理 skill。它把日常聊天里突然冒出来的想法、隐约成形的判断、还没来得及展开的念头,整理成一个个 Obsidian 兼容的 Markdown "泡泡",再通过 `[[双链]]` 把相关想法连接起来,生成一张可以浏览、拖拽和回看的灵感关系图谱。

这个项目的重点不是做一个传统笔记工具,而是让灵感在对话中自然生长:你可以随口聊一个想法,让 AI 助手帮你提炼、命名、查重、补充关联,最后沉淀到本地泡泡库里。时间久了,图谱会逐渐显露出想法之间的重复主题、隐藏线索和新的组合空间。

它适合用来记录产品点子、创作灵感、学习联想、生活观察,也适合把那些"现在还说不清,但好像有点意思"的东西先轻轻收起来。每个泡泡都是普通 `.md` 文件,可以被 Obsidian 打开和继续编辑;生成的 `图谱.html` 不依赖外部服务,离线也能查看。

每个泡泡的 frontmatter 同时保留 `created` 和 `created_at`:前者用于按天分组,后者记录精确到秒并带时区的时间戳,方便回看同一天里想法生长的先后顺序。泡泡正文里也可以保留“原句”区块,用来记录用户当时真正有判断力、有语气、有边界感的一两句原话。

灵感泡泡默认不强制项目归属。想法可以自由跨项目流动,底层结构以 `[[双链]]` 为主;项目、状态、专题都只是需要时才生成的可选视角。用户明确要求时,项目可以是一层真实文件夹,项目里的泡泡就写在那个文件夹下,但仍然保留跨项目双链。

## 可以给谁用

当前仓库已经按 Codex skill 的结构整理好,可以直接安装到 Codex 使用。它也不依赖 Codex 专有能力:核心内容是 `SKILL.md` 里的工作流说明、泡泡笔记格式和一个本地 Python 图谱生成脚本。因此,Claude、Claude Code 或其他支持自定义指令/技能文件/本地脚本的 AI agent 也可以复用这套方法。

需要注意的是,不同平台的安装方式不一样:Codex 可以直接使用本仓库的 `skill/` 目录;Claude 侧通常需要把 `SKILL.md` 和 `scripts/` 作为自定义 Skill、项目说明或上下文材料导入,并允许它读取泡泡库、运行图谱脚本。

## 位置说明

仓库里不会写入作者的本机路径。每个用户安装后,实际位置都应由正在运行的 AI 助手按当前机器环境告知:

- 项目目录:当前克隆或解压本仓库的位置。
- Codex skill 安装目录:用户自己的 Codex skills 目录下的 `灵感泡泡/`。
- 泡泡库:默认是用户主目录下的 `灵感泡泡/`,也可以迁移到用户指定的位置。
- 图谱输出文件:泡泡库里的 `图谱.html`。

如果用户想迁移泡泡库,AI 助手应该先确认目标位置,再帮用户移动或复制整个泡泡库文件夹,并在之后的图谱生成、项目管理和导出命令里使用新的 `--vault` 路径。

## 目录结构

```text
.
├── README.md
├── 图谱.html
└── skill
    ├── SKILL.md
    └── scripts
        ├── build_graph.py
        ├── manage_project.py
        └── project_server.py
```

## 使用方式

生成图谱:

```bash
python3 skill/scripts/build_graph.py . -o 图谱.html
```

启动可管理项目的图谱服务:

```bash
python3 skill/scripts/project_server.py --vault . --port 8765
```

新建项目文件夹:

```bash
python3 skill/scripts/manage_project.py create "项目名" --vault .
```

改名项目文件夹:

```bash
python3 skill/scripts/manage_project.py rename "旧项目名" "新项目名" --vault .
```

在 Codex 中使用时,skill 的运行说明位于 `skill/SKILL.md`。实际安装到 Codex 后,默认泡泡库位置会按说明使用 `~/灵感泡泡/`。

## 项目文件夹

项目是泡泡库下的一层文件夹。例如 `Fishing Agents/等待剧场.md` 会在图谱里显示为 `Fishing Agents` 项目下的泡泡。根目录下的 `.md` 仍然是自由泡泡。

图谱生成器会递归扫描项目文件夹,并在页面顶部提供 `全部项目`、`自由泡泡` 和具体项目的筛选。项目文件夹只是收纳视角,不会替代 `[[双链]]`:一个项目里的泡泡仍然可以连接到其他项目或根目录的泡泡。

如果通过 `project_server.py` 打开图谱,页面顶部的 `全部项目` 选择器可以直接筛选项目、新建项目,并通过项目右侧的小铅笔 icon 改名。普通静态服务只能查看图谱,不能创建文件夹、改名项目或把导出文件写入本地。

`导出/` 是系统生成目录,不会作为项目显示,里面的导出文件也不会被图谱扫描成泡泡。

## 导出和迁移

给 Obsidian 用时,直接把泡泡库文件夹作为 vault 打开即可。每个泡泡本来就是一个普通 `.md` 文件,关联也使用 Obsidian 兼容的 `[[双链]]`。

图谱页面右上角有三点 icon 的 `更多` 菜单,适合把内容给其他软件:

- `Markdown 索引`:把所有泡泡汇总成一个 `.md`,适合 Notion、文档软件或人工阅读。
- `灵感收成`:生成最近新增、热门主题、连接最多泡泡和幽灵泡泡小结。
- `JSON 数据`:保留标题、日期、主题、正文、关联和边关系,适合程序处理。
- `CSV 表格`:适合 Numbers、Excel、Google Sheets 做筛选和整理。

通过 `project_server.py` 使用时,导出文件会真实写入泡泡库下的 `导出/` 文件夹,页面会显示保存路径;这比依赖浏览器下载更稳定。

## 继续发散

图谱页也会帮你继续联想。主页面左下角有一个带灯泡 icon 的“继续聊聊”按钮,默认收起不遮挡图谱;每次点开后都会随机取一个泡泡,显示 5 条发散话题。面板里的“换一组”也会随机跳到另一组。每条都和对应泡泡相关,但从不同方向展开:产品化、反向质疑、跨领域类比、用户情绪、和旧泡泡组合。

点一下话题可以复制,然后直接拿去和 AI 继续聊。

## 临时专题和灵感收成

灵感泡泡可以临时整理一片想法,而不改变原始泡泡:

- 临时专题:围绕一个主题、项目、情绪或问题抽取相关泡泡。
- 多泡泡组合:把两个或多个泡泡碰撞出新方向。
- 灵感收成:图谱页右上角 `更多` 菜单可导出一份 Markdown 小结,包含最近新增、热门主题、连接最多泡泡、幽灵泡泡和组合发散建议。
